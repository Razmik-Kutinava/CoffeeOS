Отчёт: правила Cursor для CoffeeOS
Дата: 2026-04-04 (обновлено)

Сделано
- coffeeos-core.mdc — alwaysApply: true, 8 пунктов (тенант/RLS, панели, enum, миграции, секреты, объём правок, проверки, честность к схеме).
- Пять узких правил с globs (alwaysApply: false), по реальной структуре app/ и test/:
  - coffeeos-data.mdc — db/migrate, db/schema.rb, app/models
  - coffeeos-http.mdc — app/controllers, config/routes.rb
  - coffeeos-ui.mdc — app/views, app/javascript, app/assets, app/components
  - coffeeos-services.mdc — app/services
  - coffeeos-tests.mdc — test
- coffeeos-code-review.mdc — alwaysApply: false; ревью PR/диффа по запросу (тенант, RLS, схема, секреты, объём диффа). Подключай через @ или описание в чате («ревью PR», «разбор диффа»). Процесс: docs/REVIEW.md; чеклист при открытии PR: .github/pull_request_template.md.

Почему так
- Ядро даёт инварианты всегда; остальное подмешивается по открытым/редактируемым файлам — меньше шума, чем десятки правил «на каждый скилл».
- Слои разделены: данные, HTTP, UI, сервисы, тесты — совпадает с типичными PR по этому репо.

Дальше
- При частых правках только Docker/CI — опционально вынести шестой файл с globs config/**, Dockerfile, .github/**, bin/**.
